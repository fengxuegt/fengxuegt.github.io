NDK介绍

官网介绍

原生开发套件 (NDK) 是一套工具，使您能够在 Android 应用中使用 C 和 C++ 代码，并提供众多平台库，您可使用这些平台库管理原生 Activity 和访问实体设备组件，例如传感器和触摸输入。NDK 可能不适合大多数 Android 编程初学者，这些初学者只需使用 Java 代码和框架 API 开发应用。然而，如果您需要实现以下一个或多个目标，那么 NDK 就能派上用场： 1.进一步提升设备性能，以降低延迟或运行游戏或物理模拟等计算密集型应用。 2.重复使用您自己或其他开发者的 C 或 C++ 库。

NDK能够快速开发C/C++的动态库，并能够自动将so和应用一起打包成APK。

JNI介绍

官网介绍

The JNI is a native programming interface. It allows Java code that runs inside a Java Virtual Machine (VM) to interoperate with applications and libraries written in other programming languages, such as C, C++, and assembly.

JNI是一个编程接口，允许我们在java代码中调用C/C++代码。
注意：JNI是java语言的特性，与Android无关，NDK是Android的工具，与java无关。
JNI就像是在java和native之间的桥梁。

NDK是一套可以使得Android引用C/C++的工具，也就意味着NDK正是使用了java的JNI特性来实现C/C++代码的调用。

JNI的基本流程

  1. 定义一个java类A
  2. 在类A中定义一个native修饰的方法：public native String sayHello()
  3. 创建一个C/C++文件，定义一个方法，方法名为Java_类A全名sayHello拼接而成
  4. 将C/C++文件打包为一个动态链接库
  5. 在类A中通过static代码块使用System.loadLibrary()加载4中生成的动态链接库

Android ABI

这是个什么东西呢？
通俗的说就是不同的CPU支持不同的指令集，每种指令集有不同的规范，一个CPU只能读取符合他规范的机器代码。因此对于不同的CPU，代码需要编译成不同的机器码。比如一个手机的指令集是armeabi，则运行的时候会去armeabi-v7a目录寻找动态库，如果手机的指令集是AArch64，则会先去arm64-v8a目录寻找动态库，如果找不到才会去armeabi-v7a，这就是ABI兼容的体现。
如果没有指定ABI，则会默认生成四个最常用的，每个目录都有一个对应的so；

但是一般没必要四个都生成，可以使用配置生成指定的ABI。

根据ABI的兼容性，一般只需要生成armeabi-v7a的动态库就行了；

再次分析cpp文件

引入jni.h头文件


看下第四行

我们写的是CPP程序，CPP程序是支持方法重载的，为了能够区分重载的方法，CPP程序在编译的时候会把参数信息和方法名结合起来形成一个新的方法名。
extern “C” 是告诉编译器，该方法是C语言的方法。编译器不用将参数信息和方法名结合起来进行编译。
java执行native方法是通过一定的命名规则来寻找方法的。如果不加extern，那么就会将参数信息也放到方法名中，这样java就找不到对应的方法了。
再看下JNIEXPORT关键字，是jni.h里面的一个宏定义：

__attribute__表示一个属性，这里的属性就是指的visibility，属性值为default，意思是这个方法是否对外可见，这里的default表示是可见的。
其实就是确认一个方法是否放入动态链接库的一个动态方法表里，该方法可以看作是jni提供可以让java调用的方法，放入这个表就意味着java可以通过jni调用，否则就不能调用，由此控制方法接口的权限。
如果把default修改成hidden，那么程序就会执行报错。报告找不到对应的方法。
jstring，是方法的返回类型，因为java的数据类型是不能够直接在native中使用的，所以java在native代码中有对应的类型；jstring就对应java中的String；
后面的JNICALL是一个空的宏定义。它其实只是一个标记作用，用于标记这个方法可以用于JNI；

JNIEnv* env 参数，在jni中是一个核心的参数，用于链接java和c++。
JNIEnv是一个线程相关的结构体，该结构体代表了java在本线程的运行环境，通过JNIEnv可以调用到一系列的JNI系统函数，每个线程都有一个JNIEnv指针，JNIEnv只在其所在线程有效，不能在不同线程之间传递。
JNIEnv实际上就是结构体__JNIEnv，里面包含着所有java和C/C++交互的方法。

引用

在jni中，无论是从java层传递给native层，还是native层创建的java对象，本质上都是native层持有的指向JVM对象的一个引用；

局部引用

java层作为方法参数传递的java对象或者是native层创建的java对象，默认为被局部引用指向。即会在方法返回的时候自动释放引用，也可以在方法返回前手动释放。和java一样，释放引用的目的是告诉jvm这个对象这边已经不会再使用，让jvm决定是否回收。
